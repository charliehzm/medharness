# 课件 07 · 新人入职清单（7 天 / 30 天 / 90 天）

> 时长：30 分钟（看完 + 评估自己进度）
> 受众：刚加入用 MedHarness 团队的新人
> 前置：课件 01-06

---

## 学习目标

- 知道每个时间节点该达成什么
- 有一份可勾选的 checklist
- 知道遇到卡点找谁

---

## Day 1（入职日）

### 上午
- [ ] 收到 GitHub repo access + 微信群邀请
- [ ] 装好工具链（VS Code / Claude Code / .venv / pre-commit）
- [ ] 跑通本地 `bash dryrun_e2e_v2.sh`
- [ ] 读完 [README.md](../../README.md)（5 分钟上手）

### 下午
- [ ] 读完 [CLAUDE.md](../../CLAUDE.md)（红线 + Skill 索引）
- [ ] 读完 [HANDOFF.md](../../HANDOFF.md)（接手协议）
- [ ] 完成 [课件 01 · 体系总览](01-体系总览.md)

### 找谁
- 工具链卡了 → Slack #help-onboarding
- 红线不懂 → 你的 buddy（mentor）

---

## Week 1（前 7 天）

### Code 入门
- [ ] 提 1 个 typo / docs 修复 PR（micro 通道）
- [ ] 学会 GitHub PR 流程（template / review / squash merge）
- [ ] 跑一遍 `pytest tests/integration` + 看懂结果

### SOP 入门
- [ ] 完成课件 02-03（12 步 SOP + Skill 矩阵）
- [ ] 找一个 `good first issue` 标的 issue 认领
- [ ] 在 Discussions 提至少 1 个问题

### 检查
- [ ] 你能背 5 红线
- [ ] 你能说出 6 层架构每层职责
- [ ] 你的 typo PR 已 merged

---

## Day 30（一个月）

### Code 进阶
- [ ] 提 ≥ 5 个 merged PR
- [ ] 至少 1 个走 12 步主通道（非 micro）
- [ ] 至少 1 个 PR 自己当 reviewer（review 别人的）

### 合规
- [ ] 完成课件 04（Hook + PHI 防护）
- [ ] 跑过一次月度红队演练
- [ ] 在 Discussions #4 提过 / 答过至少 1 个 Hook 误判讨论

### 体系
- [ ] 完成课件 05（Memory 系统）
- [ ] 在自己负责的模块里写过 ≥ 1 个 LEGACY_FACTS / INFERENCES

### 检查
- [ ] 你能独立完成一个 12 步 SOP 流程
- [ ] 你 PR 第一次过 review 不需要重大修改
- [ ] 你的 Discussions 答复被 maintainer +1 至少一次

---

## Day 90（90 天 · 督导出口）

### Code 成熟
- [ ] 提 ≥ 20 个 merged PR
- [ ] 至少 1 个跨多文件 / 涉合规规则的大 PR
- [ ] 至少 1 个 PR 在 review 时为团队避免一次合规风险

### 体系所有权
- [ ] 完成课件 06（Skill Owner）
- [ ] 申请 own 至少 1 个 Skill
- [ ] 参加过 ≥ 1 次 community call（线下 / 线上）

### 培训能力
- [ ] 能给下一个新人讲完课件 01
- [ ] 能在 Discussions 解答 ≥ 5 个新人问题

### 检查（**90 天督导出口准则**）
- [ ] 你能独立带新人上手
- [ ] 你能独立判断"这个 change 走 12 步还是 micro"
- [ ] 你能识别 Compliance-Agent 异构性配置错误
- [ ] 你被信任：未来 maintainer 远离 1 周，你能撑住

---

## 入职反模式（不要这样）

| 反模式 | 后果 |
|---|---|
| 不读 CLAUDE.md 就开始写代码 | PR 被退 → 红线违规 |
| 跳过 Step 0 直接做实现 | 被 Hook 拦截 → 浪费一上午 |
| 第一周不提问 | 卡在小事上憋一周 |
| 学 Skill 死记硬背名字 | 实际场景识别不出 |
| 把 12 步当 ceremony 走流程 | 失去合规判断力 |
| 1 个月没提 PR | mentor 焦虑、新人怀疑 |

## 找谁矩阵

| 问题 | 找谁 |
|---|---|
| 工具链 / 装包 | Slack #help-onboarding |
| 红线疑问 | 你的 mentor / Compliance Officer |
| SOP 流程 | 你的 mentor |
| Skill 系统疑问 | Skill 对应的 Owner |
| 合规边界 | Compliance Officer / 法务 |
| 商业版咨询 | 你的 mentor 转 sales |
| 心态 / 焦虑 | 90 天督导的 1-on-1 |

---

## 课后

读完了你不是"完成"——是**开始**。
90 天后回来 self-debrief，看哪些勾上了。

每两周和 mentor 一对一 30 min，过这份 checklist。

- [ ] 我能教下一个新人
- [ ] 我已贡献至少 1 个有影响力的功能 / 修复
- [ ] 我开始有 own 的 Skill
