学完本节，你能：

---

## 1. MedHarness 一句话定位（10 min）

> **一套面向医疗数据 SaaS 公司的开源 AI Coding 落地体系。
> HIPAA + PIPL + 数据安全法 + 健康医疗数据安全指南双合规。
> 6 个月从"零散使用 Cursor"到"企业级、可审计、可演进"。**

### 不是什么（明确边界）

- ❌ **不是 Cursor / Claude Code 的替代** — 我们是研发**体系**，不是 IDE
- ❌ **不是 PHI 检测库** — 用 Microsoft Presidio，我们加医疗专属层
- ❌ **不是新 LLM 框架** — LangChain / LlamaIndex 红海
- ❌ **不是医疗器械软件** — 那是 IEC 62304 工具链
- ❌ **不是实时 AIOps 监控** — 不是我们的强项

只做**"医疗 SaaS 的 AI Coding 落地全套体系"**——一个垂直点。

---

## 2. 为什么需要这套体系（10 min）

通用 AI Coding 工具（spec-kit / OpenSpec / Cursor）解决了 60% 的问题。剩下 40% 是**医疗特有的工程纪律**：

| 问题 | 通用工具 | 医疗特有 |
|---|---|---|
| 代码质量 | spec-driven 解决 | ✅ |
| 测试覆盖 | TDD / pytest 解决 | ✅ |
| **PHI 永不裸入 prompt** | ❌ 不解决 | 🔴 必须 |
| **模型按数据分级路由** | ❌ 不解决 | 🔴 必须 |
| **6 年 WORM 审计可重放** | ❌ 不解决 | 🔴 必须 |
| **异构性 Compliance-Agent** | ❌ 不解决 | 🔴 必须 |
| **PIPL 跨境数据流监管** | ❌ 不解决 | 🔴 必须 |

这些不是工具问题，是**研发体系**问题。MedHarness 把这些"叠加"到通用工具之上。

## 3. 6 层架构（15 min）

```
L6 治理     技术委员会 + 合规委员会 + Skill Owner 制
─────────────────────────────────────────────────
L5 合规    数据分级 / PHI 脱敏 / 模型 allowlist / 审计血缘
           ↑↓ 横切，所有动作必经
─────────────────────────────────────────────────
L4 SOP     12 步主通道 + 5 步 micro 通道
─────────────────────────────────────────────────
L3 Skill   23 Skill（5 合规 + 16 通用 + 2 别名）
─────────────────────────────────────────────────
L2 Harness Orchestrator + 6 Sub-agent + Memory + 9 Hook + 8 MCP
─────────────────────────────────────────────────
L1 模型    编码 / Review / 架构 / 长文 / 脱敏 / 合规独立
```

### 每层一句话

- **L1**：哪个模型干哪个活（按数据分级 + 异构性）
- **L2**：Claude Code 的 Harness（Skill / Sub-agent / Memory / Hook / MCP）
- **L3**：23 个领域知识包（合规 + PRD + 实现 + 验证 + 审计 ...）
- **L4**：12 步 / 5 步 SOP 路径（速度 / 合规双轨）
- **L5**：横切所有层的合规闸门（哪个动作都得过）
- **L6**：人的治理（双委员会 + Skill Owner）

详 [docs/architecture/dependency-graph.md](../../docs/architecture/dependency-graph.md)。

## 4. 5 条红线（5 min）

**任何 PR / commit / 决策必自检**：

| # | 红线 | 违反 |
|---|---|---|
| R1 | L4 PHI 永不裸入 prompt | 监管处罚 |
| R2 | 模型按 allowlist 路由 | 合规违规 |
| R3 | 审计全量记录 | 监管无法重放 |
| R4 | 测试数据合规（合成 + 指纹） | 反演风险 |
| R5 | License 永久 Apache 2.0 / CC BY-SA 4.0 | 社区信任崩盘 |

详 [CLAUDE.md §1](../../CLAUDE.md)。

## 6. 现场练习（10 min）

```bash

cd medharness
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
bash dryrun_e2e_v2.sh --ci
```
