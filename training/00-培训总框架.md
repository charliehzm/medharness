# 对外团队培训总框架

> 适用：另一个产品线 / 子公司 / 战略合作伙伴 / 被并购公司 / B 端客户内部 IT
> 维护：体系负责人 + 培训师团队
> 版本：v1.0（基于 v2.2 体系经验沉淀）

---

## 1. 三条信条（培训前先说清）

### 信条 1 · 不复制文件，传递判断
"189 个文件给你，3 个月后你的体系会腐化。我们要交付的是**为什么这么做**，让你自己能维护与演进。"

### 信条 2 · 不要 v2.0，从 v2.2 学起
"如果直接抄 v2.0（完整 12 步对所有 change 一刀切），开发者 NPS 一定爆。我们已经吃过这个亏，你直接用 v2.2（含 micro 通道 + 误判抑制 + 反哺机制）。"

### 信条 3 · 合规层不可改，工程层全可改
"L5 合规层（PHI 闸门、模型路由、审计血缘）是法规决定的，不可削弱。但 L2 Harness / L3 Skill / L4 SOP 全部允许按你的业务定制。"

---

## 2. 培训 4 阶段总流程

```
阶段 A · 诊断 (1 周)
   ↓ 评估对方现状、合规边界、团队成熟度
阶段 B · 客户化 (1-2 周)
   ↓ 必改清单 + 可保留清单 + 试点边界
阶段 C · 培训 (1-2 周)
   ↓ 4 类受众分层课程 + 实操作业
阶段 D · 90 天督导 (3 个月)
   ↓ 双周答疑 + 月度复盘 + KPI 跟踪
   ↓ 第 4 月起对方自治
```

**不能跳阶段**。跳 A 直接开训 → 教材跟实际不对齐 → 失败。

## 3. 4 类受众的不同课程

| 受众 | 时长 | 目标 | 关键模块 |
|---|---|---|---|
| **决策层**<br/>CTO / 合规负责人 / 法务 / 业务负责人 | **半天** | 战略对齐 + ROI 认知 + 风险盘点 | 体系总览 / KPI 投入产出 / 合规 + 监管收益 / 失败模式 |
| **核心层**<br/>Architect / QA Lead / Skill Owner 候选 | **2 天** | 设计哲学 + 治理结构 + 主理能力 | 6 层架构详解 / 12 步 + 5 步 SOP / Skill 与 Sub-agent / 委员会治理 / Hook 与 MCP |
| **全员**<br/>开发 / QA / PM | **3 天** | 实操能力 + 红线认知 | 一遍真实 change 端到端 / 合规红线 / Skill 速查 / 申诉路径 |
| **持续运营**<br/>Memory-Curator / Skill Owner / Compliance Officer | **每月跟** | 自治演进 | 月度抽样审计 / Skill 命中率周报 / KPI Dashboard / 季度评估 |

## 5. 5 个 anti-pattern（培训师必须警惕）

| Anti-pattern | 后果 | 对策 |
|---|---|---|
| **第一天就讲技术细节**（Hook / MCP / Skill 矩阵） | 决策层听不进去，没背书 → 体系无法落地 | 第一天讲风险与 ROI；技术放第二天起 |
| **照搬我们的合规边界**（HIPAA + PIPL） | 对方可能只需 PIPL，照搬会过度繁琐 | 诊断时必问"你的监管约束清单"，按需裁剪 |
| **不让对方自己跑示例 change** | 听课没用，必须双手在键盘上 | 第三天 80% 时间实操 |
| **培训完就走** | 第 30 天 NPS 暴跌，第 60 天体系腐化 | 90 天督导必含 |
| **不培养对方主理人** | 我们一抽身，体系断 | 阶段 B 就开始指定主理人，让他贯穿全程 |

## 6. 培训"成功"的定义

**不是**：对方说"听懂了"
**不是**：对方仓库里有 189 个文件
**不是**：对方跑过一个示例 change

**是**：90 天后，对方能**自己写一份 v2.3 升级提案**——说明他们已经掌握"如何让体系跟实战反馈持续演进"。

## 7. 我作为负责人的 KPI（培训成功度量）

| KPI | 目标 |
|---|---|
| 阶段 A 诊断完成 | 1 周内 |
| 阶段 B 客户化清单签字 | 双方决策层 + 我方负责人三签 |
| 阶段 C 培训覆盖率 | ≥ 90% 应到 |
| 阶段 C 培训后小测通过率 | ≥ 80% |
| 阶段 D 第 30 天 NPS | ≥ 30 |
| 阶段 D 第 60 天 NPS | ≥ 40 |
| 阶段 D 第 90 天 NPS | ≥ 50 |
| 第 90 天对方主理人独立完成 1 次月度审计 | ✓ |
| 第 90 天对方至少有 1 次 SKILL 调优 PR | ✓ |
| 第 180 天对方至少有 1 个属于他们的 v2.3 升级 | ✓（终极成功标志） |

## 8. 培训预算参考

| 项 | 工时 |
|---|---|
| 我（负责人）亲自投入 | 阶段 A 8h + 阶段 B 16h + 阶段 C 8h（决策层） + 阶段 D 双周 1h × 6 = ~40h |
| 培训师团队（Architect + QA Lead + Harness Eng） | 阶段 C 60h + 阶段 D 双周 4h × 6 = ~85h |
| 对方团队 | 全员 24h（3 天）+ 核心层多 16h + 决策层 4h |

总成本：~125 我方工时 + ~24h × 对方全员

## 9. 不接受的培训请求

我会**拒绝**以下场景：

- 决策层不在场，直接训全员（背书缺失）
- 对方明确说"我们就抄你们的"（不诊断、不客户化）
- 时间 < 1 周（< 一天的"分享"可以，不算培训）
- 对方研发团队 > 50 人（≥ 50 需先拆 2 组）
- 对方在监管期 / 重大事件中（建议先稳定再上 AI Coding）

---

## 10. 培训师准入

担任本体系培训师必须：
- 在我们公司用过本体系 ≥ 6 个月（即从 v2.0 → v2.2 至少一轮）
- 担任过 ≥ 1 个 Skill 的 Owner
- 完整跑过 ≥ 3 个 12 步 SOP change
- 经历过 ≥ 1 次月度审计
- 由 Compliance Officer + Architect 联合提名

不允许"看过文档就上"的培训师。
