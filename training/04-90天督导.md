# 90 天督导计划（阶段 D）

> 培训结束 ≠ 体系上线。真正的考验在培训后 90 天。
> 督导节奏：双周 1 次 1 小时（我方负责人 + 培训师团队 + 对方主理人 + 必要时对方决策层）
> 第 90 天对方"接棒"，独立自治

---

## 督导节奏总览

```
Day 0      培训结束
   ↓
Day 1-14   M1 等价期 · 先锋小组 3-5 人启动
Day 15-30  M1 末 · 第一个完整 change 走完
Day 31-60  M2 等价期 · 8 Skill 上线 + 合规层先行
Day 61-90  M3 等价期 · 合规嵌入 + Hook block 切换
   ↓
Day 90     主理人独立 → 我方退出主督导
Day 90-180 月度跟进 1 次（不再双周）
Day 180    对方完成第一份 v2.3 升级提案 → 终极成功标志
```

## 阶段 D-1 · 第 1-30 天（M1 等价期）

### Day 0 · 培训结束当天
- [ ] 客户化产出物全部 commit 到对方仓
- [ ] dryrun_e2e_v2.sh 跑通（24 PASS）
- [ ] 先锋小组 3-5 人 + 主理人到位

- M1 等价期出口准则评审
- 通过 → 进 M2 等价期
- 不通过 → 延期 7 天再评

### Day 15-30 · M1 末稳定

### Day 29 · M1 总结会议（120 min）

## 主理人能力清单（Day 90 验收）

满足全部 → 接棒；任一缺失 → 督导延 30 天。

## 督导失败的 6 种信号（任一出现立即升级）

1. 你方主理人连续 2 次督导请假 / 派下属
2. 你方决策层不再出席 M1/M2/M3 总结
3. NPS 连续 2 次下降
4. 触发 P0 / P1 合规事件
5. AUDIT_BUNDLE 验证失败
6. 主理人提出"我们不打算继续了"

升级路径：我方负责人 → 你方决策层（CTO + 合规负责人 + 法务）→ 闭门会议复盘 → 决议（延长督导 / 暂停 / 改造）

## 督导中的"软"环节

技术之外的 3 件事我会**亲自做**：

1. **主理人单独咖啡** · 每月 1 次 1 小时
   - 听他抱怨（不允许下属或培训师在场）
   - 帮他识别"政治问题"（不是技术问题）
   
2. **决策层定期同步** · 每月 1 次书面简报
   - 1 页内：本月 KPI + 1 个亮点 + 1 个风险
   - 抄送你方 CTO + 合规负责人 + 法务

3. **培训师团队私下复盘** · 每月 1 次
   - 哪些课讲得不好 → 下次客户改进
   - 哪些客户化决策事后看错了 → 沉淀经验
