# 3 天课程包（阶段 C）

> 4 类受众的分层课程
> 培训师团队按本包讲；不允许"自由发挥"
> 修订需我方负责人签字

---

## 总体安排

```
Day 0 (培训前 1 周)
  ↓ 已完成阶段 A 诊断 + 阶段 B 客户化签字
  ↓ 对方主理人候选已确定
  ↓ 对方代码仓已 fork v2.2 模板 + 客户化产出

Day 1 上午 · 决策层课程（半天）  → 战略对齐
Day 1 下午 · 核心层 Day 1       → 设计哲学 + 6 层架构
Day 2 全天 · 核心层 Day 2       → SOP + Skill + Sub-agent + 治理
Day 3 全天 · 全员课程           → 实操：跑一个真实 micro-change + 一个真 feature

Day 4-5 (M2 后) · 持续运营课程  → Memory-Curator / Skill Owner / Compliance Officer
                                  每月 1 次跟进
```

## 决策层课程（Day 1 上午 · 4 小时）

> 受众：CTO / 合规负责人 / 法务 / 业务负责人

### 学习目标
- 知道这个体系是什么，**不知道**它怎么实现
- 知道投入 / 产出 / 风险
- 决议是否启动

### 课时分布

| 时段 | 内容 | 讲师 |
|---|---|---|
| 0:00-0:30 | 体系一句话 + 6 层架构 + 12 步 SOP 总览 | 我方负责人 |
| 0:30-1:00 | 我们走过 v2.0 → v2.2 的"3 次反思" | 我方负责人 |
| 1:00-1:30 | KPI 投入产出（M0-M6 真实数据） | 我方负责人 |
| 1:30-2:00 | 合规收益（监管 + 客户审计应对） | 我方 Compliance Officer |
| 2:00-2:30 | 茶歇 + 决策层私聊 | — |
| 2:30-3:00 | **失败模式预演**（你方最可能踩的 5 个坑） | 我方负责人 |
| 3:00-3:30 | 你方客户化结论复盘 | 我方负责人 + 你方 CTO |
| 3:30-4:00 | 决议 + 签字 | 我方负责人 + 你方 CTO + 你方合规 |

### 关键 slide（必讲不可省）

#### Slide 1 · 一句话定位
> "把 AI Coding 从'少数人随便用'转为'企业级、合规可审计、可度量、可演进'，6 个月落地。"

#### Slide 2 · 我们走过的 3 次反思
- v2.0 → v2.1：审计前完美主义；审计发现 24 项🔴阻断级缺漏（CLAUDE.md / sub-agent / 字段词典 ... 全没写）
- v2.1 → v2.2：完整设计但 NPS=18；实战发现 Hook 误判 66%、12 步对 micro 过重
- 教训：**你不要从 v2.0 开始，直接从 v2.2 起步**

#### Slide 3 · KPI 真实数据
| 维度 | M0 → M6 | 我方实测 |
|---|---|---|
| 单 change 周期 | ↓ 30% | 138h → 100h |
| PHI 漏出 | = 0 | 0 |
| 审计可重放 | 100% | 100% |
| 开发者 NPS | ≥ 40 | v2.1: 18 → v2.2: 40+ |

#### Slide 4 · 你方失败模式预演
基于阶段 A 诊断的针对性预警，例如：
- "你方没 KMS → M3 上线前必须采购，否则 Step 12 审计冻结跑不通"
- "你方监管是 PIPL only → 我会帮你裁掉 HIPAA-specific 部分，节省 30% 复杂度"
- "你方 25 人没指定 Skill Owner 候选 → 培训完会断"

### 决议项（决策层签字才进 Day 1 下午）
- [ ] 接受体系总框架与 6 层架构
- [ ] 接受 6 个月路线图与 KPI 目标
- [ ] 接受客户化决策清单
- [ ] 接受新岗位（Harness Engineer / Compliance Officer / Data Steward）兼任→拆分计划
- [ ] 接受 90 天督导

## 全员课程（Day 3 · 1 天）

> 受众：开发 / QA / PM
> 30 人以内一班；30+ 拆班

### 上午 (4h) · 操作课

| 时段 | 模块 |
|---|---|
| 0:00-0:30 | 体系总览（30 min 浓缩版） |
| 0:30-1:00 | 12 步 SOP + micro 5 步选哪个 |
| 1:00-1:30 | Skill 速查（21 个一张图 + 决策树） |
| 1:30-2:00 | 合规 Hook 与红线（PHI / 模型路由 / 审计） |
| 2:00-2:15 | 茶歇 |
| 2:15-2:45 | Memory 怎么用（不要让它腐化） |
| 2:45-3:30 | 误阻断怎么办（一键申诉，不要绕开） |
| 3:30-4:00 | 新人入职清单 |

### 下午 (4h) · 实操

| 时段 | 任务 |
|---|---|
| 0:00-0:30 | 分组（3-5 人一组） + 分配 sandbox change |
| 0:30-2:30 | 跑一个 micro-change 端到端（Step μ1-μ5） |
| 2:30-3:30 | 跑一个 feature change 的 Step 0-3（不要求全跑完，但走通起步） |
| 3:30-4:00 | 每组 5 min 分享 + 答疑 |

### Day 3 末小测（每人 5 题）
1. L3/L4 PHI 入 prompt 前必须先经？(phi-desensitize)
2. micro-change 必须满足的 4 个条件？(≤2 文件 / 无 L3/L4 / 无 LLM / 不在受限目录)
3. Compliance-Agent 模型必须满足？(异构于 Coder)
4. Hook 误阻断时正确的处理路径？(不绕开 / 走合规例外申请单)
5. AUDIT_BUNDLE 保留多久？(6 年)

通过分: 4/5

不能委托培训师代劳：

其他模块 → 由培训师团队执行。
