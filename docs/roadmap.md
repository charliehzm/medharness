# MedHarness 公开路线图

> 12 个月分季度交付。每季度有可量化的交付物。
> 节奏参考：开源项目 6 个月内必须有"能跑起来的 demo"，否则失去关注。

---

## Q1 · MVP 启动（M1-M3）

### 目标
**有一个完整可跑的最小骨架**，让早期用户能 fork + 跑通示例 change。

### 关键交付

| 月 | 主题 |
|---|---|
| **M1（当前）** | 仓库 + v0.1.0-alpha 发布 + 首批早期反馈 |
| **M2** | Microsoft Presidio 深度集成 · 中文医疗 recognizer · 红队 drills 全实现 |
| **M3** | Compliance-Agent + Step 0/10/12 强制 · WORM 审计闭环 |

### 出口准则
- 完整 dryrun_e2e.sh 在 macOS + Ubuntu 跑通
- 红队演练 4/4 通过（PHI recall ≥ 92%）
- 第一篇技术博客发布
- 第一次 community call（线上 1 小时）

---

## Q2 · Beta 与生态（M4-M6）

### 目标
**完整 6 层架构 + 真实 KMS 集成 + 培训方法论开源**。

### 关键交付

| 月 | 主题 |
|---|---|
| M4 | MCP 真集成（KMS / WORM / 主流模型 SDK 路由）+ 岗位拆分 |
| M5 | 培训方法论开源 + 8 节视频上线（B 站 / YouTube） |
| M6 | **v1.0.0 正式发布** + 商业版（M6 起开放）+ 行业大会演讲 |

### 出口准则
- MCP 8 件套全部"非占位"
- 第一篇行业大会演讲（如中国卫生信息大会 / HIMSS Asia）
- v1.0.0 release

---

## Q3 · 增长与标准化（M7-M9）

### 目标
**做成医疗 AI Coding 事实标准**，建立联盟 / 工作组。

### 关键交付

| 月 | 主题 |
|---|---|
| M7 | 国际化（英文文档 + LinkedIn + Hacker News）+ 招国际 contributor |
| M8 | 牵头成立"医疗 AI Coding 体系联盟" + 发布行业白皮书 + Anthropic Skill marketplace 申请 |
| M9 | 3 个公开客户案例（脱敏）+ 行业奖项申报 |

### 出口准则
- 英文文档双语完整
- 行业大会 ≥ 2 次
- 联盟 / 标准化雏形成立

---

## Q4 · 商业化加速（M10-M12）

### 目标
**财务自给 + 团队扩展 + 准备下一阶段**。

### 关键交付

| 月 | 主题 |
|---|---|
| M10 | 商业版功能完整（Dashboard + 训练模型 + SLA）+ 自助下单系统 |
| M11 | 团队扩张 + 战略评估（融资 / 独立 / 内孵） |
| M12 | **一周年回顾** + **v2.0.0 发布** + 明年路线图 |

### 出口准则
- 商业版可独立运营
- 行业标准雏形 / 联盟运转
- v2.0.0 release

## 不做的事（严守边界）

每个季度只做该季度的事：

- ❌ Q1 不做商业版（先把开源做扎实）
- ❌ Q2 不国际化（先吃中国市场）
- ❌ Q3 不融资（财务可控前不急融）
- ❌ Q4 不扩品类（不做 AI 监控 / AIOps / RAG 等周边）

## RFC 流程（影响路线图的大变更）

任何想加 / 改 / 删的路线图项目，走 RFC 流程：

1. 提案：`openspec/changes/rfc-<id>-<short-name>/proposal.md`
2. Discussions 公开讨论 ≥ 14 天
3. maintainer + Skill Owner review
4. 合并到下个 minor 版本

详 [CONTRIBUTING.md](../CONTRIBUTING.md)。
